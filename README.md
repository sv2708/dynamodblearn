This project uses Quarkus, the Supersonic Subatomic Java Framework.

If you want to learn more about Quarkus, please visit its website: <https://quarkus.io/>.

## Running the application in dev mode

You can run your application in dev mode that enables live coding using:

```shell script
./gradlew quarkusDev
```

> **_NOTE:_**  Quarkus now ships with a Dev UI, which is available in dev mode only at <http://localhost:8080/q/dev/>.

## Local DynamoDB

Start DynamoDB Local before using the cart endpoints:

```shell script
docker compose -f dynamodb/docker-compose.yml up -d
```

Then enable the local endpoint in `src/main/resources/application.properties`:

```properties
aws.dynamodb.endpoint-override=http://localhost:8000
cart.table-name=cart_details
cart.create-table-on-startup=true
```

With `cart.create-table-on-startup=true`, the application creates the `cart_details` table on startup if it is missing. The table uses `cart_id` as the string partition key.

You can also create or update the local schema explicitly:

```shell script
./dynamodb/schema/create-cart-details-table.sh
```

Cart CRUD endpoints are available under `/carts`:

```text
POST   /carts
GET    /carts
GET    /carts/{cartId}
PUT    /carts/{cartId}
DELETE /carts/{cartId}
```

OpenAPI documentation is generated by Quarkus:

```text
http://localhost:8080/q/openapi
http://localhost:8080/q/swagger-ui/
```

Create request example:

```json
{
  "cartId": "cart-1001",
  "customerId": "customer-1",
  "currency": "USD",
  "items": [
    {
      "productId": "product-1",
      "name": "Keyboard",
      "quantity": 1,
      "unitPrice": 49.99
    }
  ]
}
```

The persisted DynamoDB item uses these attributes:

```text
cart_id        string partition key
customer_id    string
status         string
items          list of cart item maps
total_amount   number
currency       string
created_at     ISO-8601 timestamp
updated_at     ISO-8601 timestamp
```

## Project structure

The cart feature is grouped by responsibility under `org.sv2708.cart`:

```text
api/          REST resources, request DTOs, and HTTP exception mappers
config/       startup infrastructure such as table creation
model/        application/domain records
repository/   DynamoDB persistence and attribute mapping
service/      business rules and cart workflow orchestration
```

## Packaging and running the application

The application can be packaged using:

```shell script
./gradlew build
```

It produces the `quarkus-run.jar` file in the `build/quarkus-app/` directory.
Be aware that it’s not an _über-jar_ as the dependencies are copied into the `build/quarkus-app/lib/` directory.

The application is now runnable using `java -jar build/quarkus-app/quarkus-run.jar`.

If you want to build an _über-jar_, execute the following command:

```shell script
./gradlew build -Dquarkus.package.jar.type=uber-jar
```

The application, packaged as an _über-jar_, is now runnable using `java -jar build/*-runner.jar`.

## Creating a native executable

You can create a native executable using:

```shell script
./gradlew build -Dquarkus.native.enabled=true
```

Or, if you don't have GraalVM installed, you can run the native executable build in a container using:

```shell script
./gradlew build -Dquarkus.native.enabled=true -Dquarkus.native.container-build=true
```

You can then execute your native executable with: `./build/dynamodblearn-1.0.0-SNAPSHOT-runner`

If you want to learn more about building native executables, please consult <https://quarkus.io/guides/gradle-tooling>.
